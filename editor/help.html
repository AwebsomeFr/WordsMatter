WordsMatter | Documentation

WordsMatter est un éditeur de texte ultra léger capable de générer du contenu Web techniquement optimisé à des fins de blogging, même en contexte d'utilisation mobile.

Créé pour promouvoir le développement de sites Internet écoresponsables, WordsMatter est une alternative viable aux Systèmes de Gestion de Contenu (CMS), peu performants et énergivores.

Pourquoi utiliser WordsMatter ?

  • WordsMatter répond strictement à un besoin: alimenter un blog. Son efficience fonctionnelle le rend plus sûr, plus rapide et bien plus indolore pour l'environnement que les solutions traditionnelles.
  • WordsMatter fonctionne en mode hors-ligne sur la majorité des navigateurs Web, il peut être ajouté en tant que Progressive Web App (PWA) à tout terminal compatible et son interface responsive permet de l'emporter partout.
  • WordsMatter n'utilise aucune dépendance. De fait, son intégrité ne tient qu'à lui et à ses utilisateurs et n'est pas conditionnée à d'incessantes mises à jour.
  • WordsMatter est libre de droits et distribué gratuitement.

Sommaire

  1. Utilisateurs : maîtriser l'Application WordsMatter

    Menu principal

    Cliquez sur pour dévoiler les options suivantes :

    • : Enregistrer votre post — un seul à la fois — dans la mémoire locale du navigateur Web pour être automatiquement rechargé au lancement de WordsMatter.
    • : Supprimer de la mémoire locale du navigateur Web le post sauvegardé.
    • : Mettre en ligne le post en cours d'édition sur votre blog. Si un post portant le même nom existe déjà, WordsMatter vous proposera de l'écraser.
    • : Éditer ou supprimer un post publié sur le blog.
    • : Exporter le contenu de votre post vers un fichier .txt. Utilisez ensuite l'option "↓ Importer depuis un fichier .txt" pour rouvrir le fichier.
    • : Importer le contenu d'un fichier .txt créé avec WordsMatter avec l'option "↑ Exporter vers un fichier .txt".
    • : Basculer entre le thème clair, plus lisible en extérieur, et le thème sombre, moins agressif pour les yeux.
    • : Définir la largeur de travail de WordsMatter (minimum : 360px / maximum : 1280px). Cette fonctionnalité est indisponible sur les terminaux dont la largeur d'écran est inférieure à 768px.
    • : Définir la hauteur de travail allouée à l'espace de rédaction.
    • : Affiche la présente documentation.

    Syntaxe WordsMatter

    Avec WordsMatter, votre post est ainsi découpé :

    • Un titre principal.
    • Une introduction optionnelle.
    • Autant de sections que nécessaire.

    WordsMatter utilise sa propre syntaxe pour mettre en forme du contenu ; voici deux généralités à ce sujet :

    • Identifiant : toute mise en forme commence toujours par un chiffre OU une lettre indiquant à WordsMatter quel type de mise en forme doit s'appliquer au contenu.
    • Conteneur : l'identifiant est toujours suivi d'au moins un couple de parenthèses autour du contenu à formater. Il ne doit y avoir aucun espace entre l'identifiant et le conteneur.

    Certaines mises en forme imbriquent plusieurs couples de parenthèses, mais rassurez-vous : le menu d'édition propose un accès assisté à chacune d'elles.

    • <h1> : Titre du post

      Remplissez le champ textuel dont le label est "Titre du post". Ce titre, d’une importance capitale en matière de référencement (SEO), devrait présenter le sujet du contenu. Cet élément est requis pour publier un post.

    • <h2> : Titre de section

      Remplissez le champ textuel dont le label est "Titre de section" correspondante. Ce titre, important en matière de référencement, devrait introduire la section qu'il précède.

    • <h3> / <h4> / <h5> / <h6> : Sous-titre de section

      Écrivez, par exemple, 4(votre sous-titre) ou utilisez le bouton pour transformer votre sélection en sous-titre de niveau 4. En l'absence de sélection, vous serez invité à saisir du texte. Vous pouvez utiliser de 3(titre de plus grande importance) à 6(titre de moindre importance). Veillez à hiérarchiser vos sous-titres.

    • <p> : Paragraphe

      Pressez la touche [Enter] une fois.

    • <br/> : Retour à la ligne

      Tapez deux espaces en fin de ligne avant de presser la touche [Enter] une fois. Un retour à la ligne ne crée pas de nouveau paragraphe, mais permet d'aérer des idées, sans pour autant les isoler.

    • <strong> : Texte important

      Écrivez s(votre texte important) ou utilisez le bouton pour transformer votre sélection en texte important. En l'absence de sélection, vous serez invité à saisir du texte. Assurez-vous que ce texte est important dans son contexte.

    • <em> : Texte avec emphase

      Écrivez e(votre texte emphasique) ou utilisez le bouton pour transformer votre sélection en texte emphasique. En l'absence de sélection, vous serez invité à saisir du texte. Assurez-vous que ce texte possède une signification particulière dans son contexte.

    • <ol> : Liste ordonnée

      Écrivez o(((premier élément))((deuxième élément))((énième élément))) ou utilisez le bouton pour créer une liste ordonnée. Vous serez alors invité à saisir les uns après les autres les éléments qui composent votre liste. Une fois votre liste terminée, annulez la saisie (touche [Echap], bouton Annuler...) pour valider les entrées précédemment saisies.

    • <ul> : Liste non ordonnée

      Écrivez u(((premier élément))((deuxième élément))((énième élément))) ou utilisez le bouton pour créer une liste non ordonnée. Vous serez alors invité à saisir les uns après les autres les éléments qui composent votre liste. Une fois votre liste terminée, annulez la saisie (touche [Echap], bouton Annuler...) pour valider les entrées précédemment saisies.

    • <a> : Lien hypertexte

      Écrivez a(libellé du lien(URL)) ou utilisez le bouton pour insérer un lien hypertexte. Vous serez alors invité à renseigner le libellé du lien puis son URL. Le libellé sera exploité par les moteurs de recherche et certains utilisateurs en situation de handicap ; il devrait donc donner une indication claire sur le contenu vers lequel il pointe.

    • <img> : Image

      Écrivez i(description alternative(URL)) ou utilisez le bouton pour insérer une image. Vous serez alors invité à renseigner une description alternative à l'image puis l'URL à laquelle cette ressource est disponible. La description sera exploitée par les moteurs de recherche, certains utilisateurs en situation de handicap, et apparaîtra en lieu et place de l'image si cette dernière ne peut être affichée (adresse invalide, ressource indisponible...). Elle devrait donc être aussi descriptive et concise que possible.

    • <figure> + <figcaption> : Image avec légende

      Écrivez f(légende(description alternative(URL))) ou utilisez le bouton pour insérer une figure (image avec une légende). Vous serez alors invité à renseigner la légende de l'image, une description alternative puis l'URL à laquelle cette ressource est disponible. La description sera exploitée par les moteurs de recherche, certains utilisateurs en situation de handicap, et apparaîtra en lieu et place de l'image si cette dernière ne peut être affichée (adresse invalide, ressource indisponible...). Elle devrait donc être aussi descriptive et concise que possible.

    Pour obtenir les meilleures performances possibles en termes de référencement (SEO on-page), respectez le rôle sémantique des éléments HTML introduits ci-dessus et produisez du contenu de qualité à destination de vos utilisateurs.

  2. Développeurs : installer et configurer l'API WordsMatter

    Technologies

    • Côté client, l'Application WordsMatter repose exclusivement sur HTML, CSS et JavaScript.
    • Côté serveur, l'API WordsMatter est codée en PHP natif.
    • Aucune base de données n'est requise pour utiliser WordsMatter.

    Configuration et politique de sécurité

    L'archive WordsMatter est divisée en deux parties :

    • Le dossier editor contient l'Application WordsMatter et la présente documentation. Il doit être téléchargé ou installé sur le ou les terminaux de l'utilisateur et ne doit en aucun cas être hébergé sur le serveur.
    • Le dossier publisher contient l'API WordsMatter. à placer en lieu sûr sur le serveur (voir explications plus bas).

    Les fichiers :

    • /editor/scripts/config.js
    • /publisher/config.php

    Contiennent tous deux des constantes qu'il convient de personnaliser AVANT TOUT LANCEMENT de WordsMatter afin de créer une configuration unique et sécurisée. Si vous avez lancé WordsMatter avant d'avoir édité ces fichiers, assurez-vous de supprimer le dossier blog généré à la racine de votre projet avant de commencer ici.

    • config.js :
      • EDITOR_ID est un identifiant unique attribué à l'éditeur et stocké en clair. Pour des raisons évidentes de sécurité, cette valeur devrait être la plus complexe possible.
      • PUBLISHER_URL doit pointer vers l'adresse du site Web où le dossier de l'API est hébergé. Ce dossier doit être renommé (explications plus bas).
    • config.php :
      • EDITOR_ID doit être égale à la constante du même nom décrite ci-dessus.
      • INPUT_FOLDER doit pointer vers l'adresse où les posts rédigés seront sauvegardés pour modification éventuelle (format .txt). Ce dossier sera automatiquement créé par WordsMatter lors de sa première connexion avec l'API.
      • OUTPUT_FOLDER doit pointer vers l'adresse où les posts formatés seront exportés pour publication sur le site Web (format HTML, par défaut). Ce dossier sera automatiquement créé par WordsMatter lors de sa première connexion avec l'API.
      • LIST_OF_POSTS doit pointer vers un fichier au format .json où seront listés les posts avec leur titre, leur nom de fichier et leur dernière date publication. Ce fichier sera automatiquement créé par WordsMatter lors de sa première connexion avec l'API.

    Parce que garder la maîtriser des opérations en lien avec le serveur est capital, voici les points à retenir :

    • Requêtes CORS : Pour que l'Application puisse échanger avec l'API, les requêtes CORS (Cross-origin resource sharing) doivent être autorisées par le serveur. Le dossier publisher contient donc un fichier .htaccess configuré en ce sens. Par précautions, sauf besoins contraires, l'acceptation des requêtes CORS devrait se limiter à ce seul dossier. Le fichier .htaccess est un fichier caché. Assurez-vous de l'inclure lorsque vous déplacez des fichiers ou de sélectionner le dossier publisher tout entier.
    • Identification de la source et protection de la cible : Aucun mot de passe ne sera jamais demandé à l'utilisateur ni par l'Application ni par l'API. Ces dernières sont configurées pour échanger dès lors que l'identifiant d'éditeur établi dans les fichiers config.js et config.php concordent. Pour éviter les attaques de force brute liées à une URL d'API prévisible, il est donc impératif de renommer le dossier publisher dans le but de créer une adresse personnalisée et impossible à deviner. Dans la mesure où ce nom de dossier doit pouvoir constituer une URL valide, cela exclut l'utilisation de certains caractères. Une solution peut être de combiner minuscules, majuscules et chiffres (sans accents) dans la longueur. Par exemple, un intitulé de 100 caractères tel que "KguZ5n9vnXh4saDpe65LcmgzybjLcTG9wdqveEkwWvFx5eZLpwVXrpFnQwEvSfc5fEh2EeAbhpFDWd3MPAbNMFNXYASHHr4CACcv" pourrait donner lieu à une URL imprévisible telle que "https://votre-site-.fr/KguZ5n9vnXh4saDpe65LcmgzybjLcTG9wdqveEkwWvFx5eZLpwVXrpFnQwEvSfc5fEh2EeAbhpFDWd3MPAbNMFNXYASHHr4CACcv/". Attention : pour que cette protection soit efficace, il faut interdire l'exploration de l'arborescence du site (exemple : .htaccess à la racine du site contenant la directive Options -Indexes).

    Considérations

    WordsMatter est tout à fait fonctionnel en l'état, cependant :

    • Le modèle de page fourni par l'API est basique et dépourvu de CSS. Ce modèle, décrit dans le fichier template.php, est un bon point de départ pour créer un site Internet écoresponsable et unique ; à vous de vous l'approprier. Créer un sous-dossier dédié ici ou ailleurs est tout à fait envisageable (et même préférable) du moment que correspondance est faite avec fichier releasePost.php.
    • WordsMatter ne génère pas de sommaire de blog prêt à l'emploi. Il existe bien des manières de procéder : à vous d'implémenter la vôtre. Le fichier list-of-posts.json créé dans le dossier blog que l'Application utilise pour lister les posts peut facilement être exploité dans ce but.
    • WordsMatter ne produit aucun sitemap.

    Notes de version 2.0.0

    Voici les changements majeurs apportés par cette nouvelle version :

    • Nouveau nom et nouveau logo : TextMatters n'est plus ; vive WordsMatter !
    • L'architecture a été repensée pour offrir de meilleures performances, des mécanismes de défenses solides, et permettre une meilleure appréhension du code source par les développeurs. Les scripts ont été entièrement réécrits afin de limiter le code legacy et HTML est désormais utilisé pour construire le squelette de l'application (contre JavaScript auparavant).
    • Les fonctionnalités essentielles qui manquaient à TextMatters, comme la possibilité de modifier ou supprimer un post publié, ont été implémentées dans WordsMatter. Les fonctionnalités de redimensionnement de l'espace de travail ont quant à elles été revues pour offrir davantage de contrôle.
    • Une politique verbeuse de gestion d'erreurs a été mise en place à travers une boîte de dialogue personnalisée ayant des interactions riches avec l'utilisateur et la configuration utilisateur est testée à chaque lancement de l'Application pour ne proposer que les fonctionnalités qui peuvent effectivement être exploitées.
    • L'interface utilisateur a été retouchée pour offrir un confort d'utilisation accru (meilleurs contrastes, optimisation du layout...).
    • Le chargement différé sur les images (lazy loading) est automatiquement appliqué aux images lors de la génération du post grâce à l'attribut natif loading.
    • La documentation a été réécrite pour faciliter son appropriation par les utilisateurs et les développeurs.
    • Les fontes personnalisées ont été supprimées au profit de Web safe fonts.
    • La prévisualisation du code généré depuis l'Application, dont l'intérêt était limité, a également été retirée.
    WordsMatter version 2, anciennement TextMatters, ayant été entièrement réécrit depuis la version 1, la compatibilité ascendante n'est pas assurée.
